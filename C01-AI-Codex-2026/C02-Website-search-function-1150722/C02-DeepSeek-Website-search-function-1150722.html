製作目的

為 AI學苑 網站建立一個「全站搜尋網頁」。使用者輸入關鍵字後，可找出關鍵字出現在哪些網頁，並直接點選結果前往該頁面。

大量 HTML 網頁與學習紀錄
涵蓋 AI・閱讀・退休・網頁設計・Python・提示詞
免費・靜態・無需付費服務

完成後可透過 /search.html 開啟，導覽列也新增了搜尋圖示入口。

採用的方法

靜態搜尋索引 方式，而非瀏覽器即時掃描所有檔案。

所有 HTML Node.js 索引產生 site-search-index.json search.html 結果連結至網頁
免費，無需資料庫或後端
可在 Live Server 直接使用
可部署到 Netlify / GitHub Pages
搜尋資料只在使用時下載，不影響首頁速度

  • 標題命中優先排序
  • 顯示摘要

tools/build-search-index.mjs
  • 掃描所有 .html

  • 擷取 title, 路徑, 正文
  • 排除 script, style, 註解
  • 產生索引檔
data/site-search-index.json
  • 收錄 224 個 HTML 網頁
  • 大小約 1.72 MB
  • 僅在進入搜尋頁時下載

另有導覽列 CX01-nav.html 新增放大鏡圖示。

製作流程

  1. 統計網站網頁數量 – 確認約 224 個 HTML，避免直接掃描造成緩慢。
  2. 建立索引產生程式 – 使用 Node.js 讀取每個 HTML，轉換為搜尋文字。
  3. 建立搜尋頁 – search.html 提供手機可用介面，顯示標題、路徑、摘要。
  4. 建立前端搜尋邏輯 – site-search.js 依序搜尋標題、路徑、正文。
  5. 建立索引檔 – 執行 node tools/build-search-index.mjs 產生 JSON。
  6. 加入導覽列入口 – 新增放大鏡圖示，從任何頁面前往搜尋。
終端機指令範例： node tools/build-search-index.mjs

日後維護方式

每當新增、修改或刪除 HTML 網頁後，都必須重新建立搜尋索引：

新增/修改網頁 儲存 HTML 執行 build 重新整理 search.html 測試新內容

每次執行：node tools/build-search-index.mjs

測試方式

建議測試關鍵字：

AI Python 退休 提示詞 閱讀
  • 搜尋頁可以正常開啟
  • 輸入關鍵字後有結果
  • 結果數量合理
  • 點選標題可前往正確網頁
  • 手機寬度下操作正常
  • 新增的文章可以被搜尋到

常見問題與除錯

尚未產生 data/site-search-index.json。請執行 node tools/build-search-index.mjs。

通常是新增 HTML 後未更新索引。請重新執行 node tools/build-search-index.mjs 並重新整理搜尋頁。

可能是 HTML 檔已改名或移動。重新建立索引即可。

瀏覽器會限制本機檔案讀取 JSON。請使用 Live Server 開啟網站。

若未來超過 5~10 MB，可考慮依主題分類索引，或改用 Pagefind 等工具。
